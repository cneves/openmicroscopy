This section details the structure and dynamics of the execution infrastructure. This is a set of classes that collaborate to:

  • Factor out the implementation logic common to all calls within the views.
  • Configure and execute commands.
  • Monitor execution and relay execution events to the invoker.

 

All this is described in the following diagrams. We start with a structural diagram to present the key classes, their relationships and roles.

 
Fig 5-1: Structure.

The next diagram analyzes a concrete call to a view to exemplify the pattern followed by all asynchronous calls in the various views. The call is mapped onto a command, the command is transferred to a processor for asynchronous execution, a handle to the call is returned to the invoker.

 
Fig 5-2: Invocation.

As the computation proceeds asynchronously, the processor sends execution events. This last diagram shows how they're captured and relayed to the listener specified by the invoker.
