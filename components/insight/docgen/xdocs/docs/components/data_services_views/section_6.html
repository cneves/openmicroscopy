A call in a view maps to a command (a CompositeBatchCall object) which is maintained by a BatchCallTree. Concrete subclasses of BatchCallTree build the command by aggregating single calls to the data services -- each such call is represented by a concrete BatchCall. Typically, a command is just a plain sequence of calls to the data services. However, BatchCall and CompositeBatchCall provide a composite structure which can be used to create more generic (and complex) computation trees where leaves (instances of concrete BatchCall's) are actual calls to the data services and internal nodes are CompositeBatchCall objects that are used to build the tree structure.
