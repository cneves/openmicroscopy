The DataViewsFactory needs to be initialized before any concrete BatchCallTree is created. The reason for this is that BatchCallTree's constructor needs to cache a reference to the registry so that concrete subclasses can access it later.

The DataServicesFactory (in env.data) takes care of this initialization task during the container's start-up procedure by calling DataViewsFactory.initialize(Container).
