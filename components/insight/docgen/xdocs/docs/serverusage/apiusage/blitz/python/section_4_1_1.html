Interpreting the Slice definitions in API.ice



This chapter is going to give an overview of the slice language as it pertains to
intepreting the methods available in the OMERO.Blitz server. A more detailed manual
on the Slice language can be found at 

Slice Chapter -- ICE Manual.



The API.ice file contains all of the service alls that are exposed to the client.
Slice files are broken into separate sections.



    

  1. 
	Module -- This is the namespace definitions in the Slice language.

    2. 

  2. 
	Interface -- An interface definition can be thought of as the equivalent 
	of a Java interface, and of operation definitions as (virtual) member 
	functions. In an interface only the operation definitions are allowed to 
	appear. You may not define types, exceptions or datamembers inside an interface.
	This does not mean that an object which is defined by an Interface is stateless.

    3. 

  3. 
	Class -- Similar to an interface a class can have exception, datatype and datamembers 
	defined in it.

    4. 

  4. 
	Struct -- A Stuct defines a datatype which may be used to pass intformation 
	via the ICE interface.
